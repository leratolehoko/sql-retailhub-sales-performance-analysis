# RetailHub Sales & Performance Analysis

## Project Overview

This project demonstrates the use of SQL to analyze retail sales performance using a relational database built in MySQL.

The database consists of five interconnected tables representing stores, employees, customers, products, and orders. SQL was used to answer business questions related to revenue, employee performance, customer spending, product categories, and sales trends.

---

## Database Structure

The project contains the following tables:

- Stores
- Employees
- Customers
- Products
- Orders

Relationships were established using Primary Keys and Foreign Keys to maintain data integrity.

## Business Questions Answered

- Which store generated the highest revenue?
- Which employee generated the highest revenue?
- Which customer spent the most?
- Which product category generated the highest revenue?
- Which city generated the highest revenue?
- Which employee processed the most orders?
- What is the average order value for each store?
- Which products are priced above the average?
- Which customers spent above the average customer spending?
- How can stores and products be classified using CASE statements?
- What is the monthly revenue generated?
- Which products have never been sold?
